\subsection{Detailed Description}
\subsubsection*{template$<$int W, int K, int S = Q\-U\-E\-U\-E\-S\-I\-Z\-E$>$class Cir\-Sched$<$ W, K, S $>$}

Schedule lookup requests in a circular pipeline. 

In a circular pipeline, a search request may start searching from any pipe stage (dependent on the location of the target subtrie) and wrap around the pipeline to finsh the task.


\begin{DoxyParams}{Parameters}
{\em W} & at most W search steps for a request \\
\hline
{\em K} & number of pipe stages \\
\hline
{\em S} & size of request queue per stage\\
\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
K $>$= W. That is, the number of pipe stages is no fewer than the height of the tree (the number of search steps) 
\end{DoxyNote}
