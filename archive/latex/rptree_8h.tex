\subsection{Detailed Description}
Definition of an I\-P lookup index based on prefix tree. Copyright (c) 2016, Sun Yat-\/sen University All rights reserved

The lookup index consists of three parts\-: a fast lookup table that contains route information for prefixes shorter than U bits, a table that consists of 2$^\wedge$\-U entries and each entry stores a pointer to the root of a prefix tree, and a set of binary trees.
