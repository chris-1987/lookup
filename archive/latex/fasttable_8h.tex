\subsection{Detailed Description}
definition of a fast table for searching short prefixes Copyright (c) 2016, Sun Yat-\/sen University, All rights reserved

Build a fast table for prefixes not longer than U bits. For each prefixes shorter than U bits, we expand them to U bits by padding 0. All the prefixes with a same expanded form are gathered together in the table. To insert/delete a prefix into/from the table, we expand the prefix to U bits by padding 0 and use $<$expanded\-Prefix$>$ as an index to find the set of prefixes that have a same expanded form and use its length to check whether or not it exists.
